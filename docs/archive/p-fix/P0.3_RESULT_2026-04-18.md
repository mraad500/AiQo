## Behaviour changes worth noting

1. **stepsBucketSize: 50 → 500.** Prior granularity was tight enough to fingerprint trends across sessions. Now aligned with the free-text bucket. Gemini sees slightly rounder step numbers; no tests rely on the old value.
2. **Int bucketing direction: round → floor** (across both free-text and structured paths). Floor gives a strict upper bound on leaked information and matches Swift's integer-division idiom used in the helpers. Float buckets are unchanged (still round-to-nearest-even).
3. **`CloudBrainService` consent flow:** replaced the leading `try await AICloudConsentGate.requireConsent()` call with an inline `AIDataConsentManager.shared.hasUserConsented` check (combined into the existing tier/memories MainActor hop). Behaviourally identical when `presentIfPossible == false`, which was the original argument. Benefit: we can emit an audit entry with the correct tier label before throwing.

## Audit entry shape

```swift
AuditLogger.Entry(
    id: UUID,
    timestamp: Date,                 // request start
    destination: String,             // "gemini-3-flash-preview" | "gemini-2.5-flash" | "none" (consent-denied)
    tier: String,                    // "max" | "pro" | "none"  (from SubscriptionTier.auditLabel)
    promptBytes: Int,                // UTF-8 byte count of sanitized prompt inputs
    responseBytes: Int,              // UTF-8 byte count of reply.message (0 on failure)
    latencyMs: Int,
    consentGranted: Bool,
    sanitizationApplied: Bool,       // always `true` for successful requests (false only for consent-denied)
    purpose: String,                 // RequestPurpose.rawValue
    outcome: Outcome                 // success | failure | sanitizerBlocked | consentDenied | rateLimit
)
```

Persisted to `~/Documents/brain_audit.log.json` with `.atomic` writes. No network upload, no PII, no prompt/response bodies.

## Known deferred items

- **`Brain/05_Privacy/` folder structure.** The spec placed `AuditLogger.swift` at `AiQo/Features/Captain/Brain/05_Privacy/AuditLogger.swift`. That folder does not exist in the codebase; all Captain infra lives flat under `AiQo/Features/Captain/`. I used the flat path to stay consistent with existing convention. Moving to a namespaced subtree is a separate refactor.
- **BrainDashboard debug surface.** `AuditLogger.shared.recentEntries(limit:)` and `clear()` are ready for a viewer; the viewer UI itself is not part of this prompt and was not added. Dev can inspect via LLDB: `po await AuditLogger.shared.recentEntries(limit: 20)`.
- **Rate-limit outcome** is defined in the enum but not emitted — there's no rate limiter on `HybridBrainService` yet. Added for future use.
- **`sanitizerBlocked` outcome** is also defined but unused. The sanitizer currently doesn't refuse inputs — it only redacts. Leaving the case in for future when we add "refuse if prompt cannot be safely sanitized" logic.
- **Single-letter Arabic unit `د`** (sometimes used for minutes) intentionally omitted from the duration regex to avoid catastrophic false positives on common Arabic words.
- **`\b` word boundaries** only applied to English duration units. Arabic word boundaries don't work reliably with NSRegularExpression's ICU defaults, and Arabic unit words are distinct enough that false positives are rare.
