## Concurrency Profile

- `EpisodicStore` and `SemanticStore` are actor-isolated.
- All externally-used store APIs are async at the call site.
- Each store operation creates its own `ModelContext` from the configured `ModelContainer`; no `ModelContext` escapes actor boundaries.
- Store reads return typed snapshot structs instead of leaking raw SwiftData context handles.
- Transition shadow writes from `MemoryStore` run in background `Task(priority: .utility)` blocks so the existing write path remains the source of truth during coexistence.
