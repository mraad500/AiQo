## Test baseline (P2) — 5 failures, ALL stale tests (app code is correct)

**Root-cause established by reading code + git. None is a functional regression.**

1. **`TierGateTests.testMemoryRetrievalDepthLimits`** — test expects max=10/pro=25; code
   (`TierGate.swift:141`) intentionally has **max=18/pro=40**. Stale assertion.
2. **`TierGateTests.testSemanticFactLimits`** — test expects none=0/max=200/pro=500; code
   (`TierGate.swift:154`) intentionally has **none=120/max=600/pro=1200**, with a dated comment
   ("product decision 2026-05-18": free is non-zero so Saved-Memories+reminders persist all-tier).
   Matches memory `project_saved_memories_reminders`. Stale assertion.
3. **`TierGateTests.testCappedMemoryFetchLimitRespectsTier`** — depends on the same ceilings
   (max cap now 600 not 200; none floor now 120 not 0). Stale assertion.
4. **`IngredientAssetCatalogTests.testAllMappedIngredientAssetsExistInFoodPhotosCatalog`** and
5. **`IngredientAssetLibraryTests.testAllMappedIngredientAssetsExistInFoodPhotosCatalog`** —
   both assert that 67 `ing_*.imageset` photos exist under `Assets.xcassets/Food_photos/`. That
   catalog (141 files, many 1.5 MB+ PNGs) was **deliberately deleted in commit `29db40f`**; the
   Kitchen UI moved to **emoji** (`IngredientIconView`). The photo-loading APIs
   (`IngredientAssetCatalog.uiImage/mappedAssetNames`, `IngredientAssetLibrary.*`,
   `IngredientLocalAssetView`) now have **zero live callers** — pure dead code. Stale tests.

**Fix plan (P5, first):** update the 3 TierGate assertions to the intentional current values;
drop the 2 obsolete photo-existence tests; remove the dead ingredient-photo APIs (keep emoji
`IngredientIconView` + `IngredientCatalog` alias matching + the alias-resolution test). Re-run → green.
(Founder option, deferred: re-introduce *optimized* ingredient photos later for extra premium feel.)
