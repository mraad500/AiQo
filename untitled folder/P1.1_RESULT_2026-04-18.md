## Stub content conventions

Three minimal stub shapes used depending on the kind annotation in the spec:

- **Namespace enum** тАФ `public enum Foo {}`
- **Final class (no singleton)** тАФ `public final class Foo {}`
- **Final class (singleton)** тАФ `public final class Foo { public static let shared = Foo(); private init() {} }` (used for types spec marked as `final class (shared)`)
- **Actor** тАФ `public actor Foo {}` тАФ with the `.shared` singleton form only for `BrainBus` (spec-provided example)
- **Struct** тАФ `public struct Foo {}`
- **SwiftUI View** тАФ `public struct BrainDashboard: View { public init() {}; public var body: some View { Text("Brain Dashboard") } }` тАФ the only stub that imports `SwiftUI` instead of `Foundation`

Three spec-provided example stubs (`BrainBus`, `FeatureFlags`, `TierGate`) were written verbatim per the prompt.

### @Model class deferral

`Brain/02_Memory/Models/` contains 5 stubs (`EpisodicEntry`, `SemanticFact`, `ProceduralPattern`, `EmotionalMemory`, `Relationship`) that the spec annotates as `@Model class`. I intentionally did **not** add `@Model` attributes. SwiftData `@Model` requires at least one stored property to compile cleanly, and the prompt's hard rule "Do not modify SwiftData schemas" made a schema commitment premature. Current form: empty `public final class Foo {}` with a `TODO(P2.1): convert to @Model with SwiftData schema` comment.
