## 1. Stores created

### `ProceduralStore`
- File: `AiQo/Features/Captain/Brain/02_Memory/Stores/ProceduralStore.swift`
- Actor singleton; `configure(container:)`; upsert reinforces existing patterns and clamps `strength` to `[0, 1]`; `recordException` weakens by `0.05` and clamps at `0`; append-only `observationLog` capped at 100 via `contextualDataJSON`.
- Exposes snapshot type `ProceduralPatternSnapshot` so callers don't hold `@Model` references.

### `EmotionalStore`
- File: `AiQo/Features/Captain/Brain/02_Memory/Stores/EmotionalStore.swift`
- Actor singleton; `record`, `unresolvedEmotions(olderThan:minIntensity:)`, `markResolved(_:)`, `emotions(kind:since:)`, `deleteAll`.
- `unresolvedEmotions` filters by `resolved == false && intensity >= min && date <= cutoff`, sorted by intensity desc.
- Snapshot type `EmotionalMemorySnapshot` exposes decoded `associatedFactIDs` + `bioContext`.

### `RelationshipStore`
- File: `AiQo/Features/Captain/Brain/02_Memory/Stores/RelationshipStore.swift`
- Actor singleton; `upsert(name:kind:sentimentDelta:)` — sentiment clamped to `[-1, 1]`, weight clamped to `[0, 1]`.
- `recentlyMentioned(in:within:)` filters by `lastMentionedAt >= cutoff` in SQL, then name-substring match in Swift.
- Snapshot type `RelationshipSnapshot`.
