**Purpose:** Feed into P0.2 emergency fix
**Mode:** READ-ONLY RECON — no source files modified

## 1. Executive Summary

Five most dangerous findings, ranked by blast radius:

1. **Arabic Captain API leaks raw, unsanitized prompt text to an external endpoint.** `CaptainIntelligenceManager.generateArabicAPIReply` ([CaptainIntelligenceManager.swift:274-313](AiQo/Features/Captain/CaptainIntelligenceManager.swift:274)) POSTs the caller's prompt as-is (`{"text": userInput}`) to `CAPTAIN_ARABIC_API_URL`. There is no call to `PrivacySanitizer`. Every upstream caller that routes Arabic through this path leaks whatever it stuffed into the prompt.
2. **Workout summary prompts embed raw HealthKit numbers and route through that same Arabic API.** `AIWorkoutSummaryService.buildPrompt` ([NotificationService.swift:850-909](AiQo/Services/Notifications/NotificationService.swift:850)) builds an Arabic prompt with `calories`, `averageHR bpm`, `distance km`, HR-zone percentages, and minutes, then calls `intelligenceManager.generateCaptainResponse(for: prompt)` ([NotificationService.swift:629](AiQo/Services/Notifications/NotificationService.swift:629)). Because the prompt contains Arabic, CIM's auto-route sends it to the Arabic API — raw bpm and calories land in the remote log.
3. **Background "coach nudge" translation leaks raw steps and heart rate to Gemini.** `SmartNotificationScheduler.englishCoachMessage(...)` ([SmartNotificationScheduler.swift:555-579](AiQo/Core/SmartNotificationScheduler.swift:555)) interpolates `"around \(heartRate) bpm and you're at \(context.steps) steps"` and then `translateToFriendlyIraqiArabic(...)` ([SmartNotificationScheduler.swift:720-737](AiQo/Core/SmartNotificationScheduler.swift:720)) hands that text to `CoachBrainLLMTranslator.translate` which POSTs it to Google's Gemini endpoint.
4. **Kitchen plan generator ships `userGoal`, `fridgeItems`, and `userMessage` to the Arabic API with zero sanitization.** `KitchenPlanGenerationService.buildKitchenChatPrompt` / `buildStructuredPlanPrompt` ([KitchenPlanGenerationService.swift:96-166](AiQo/Features/Kitchen/KitchenPlanGenerationService.swift:96)) both flow through `intelligenceManager.generateCaptainResponse` with `forcedRoute: .arabicAPI` for any Arabic-detected message.
5. **`PrivacySanitizer` is blind to any HealthKit vitals outside `CaptainContextData.{steps, calories}`.** Heart rate, HRV, SDNN, RHR, VO2Max, sleep hours, distance, zone%, and workout type are not bucketed or normalized anywhere. `CloudBrainService` is the *only* correctly-wired caller (it passes only contextData bucketing); any free-text prompt that mentions these vitals leaks with full precision.

## 3. Raw HealthKit Leak Inventory

### Leak #1 — Arabic API funnel (the root leak)
- **Location:** [AiQo/Features/Captain/CaptainIntelligenceManager.swift:274-313](AiQo/Features/Captain/CaptainIntelligenceManager.swift:274)
- **Trigger:** Any `generateCaptainResponse` call whose cleaned input is detected Arabic, OR any call with `forcedRoute: .arabicAPI`. That includes the Captain chat Arabic path (but Chat uses BrainOrchestrator, not CIM, so Chat doesn't reach here in practice), Kitchen plan generation, Kitchen chat reply, NotificationService inactivity & workout prompts when Arabic-localized.
- **Data leaked:** whatever is in the caller's prompt string — in practice: `calories: Int`, `averageHR: Int bpm`, `distance: km (formatted)`, `zone2/below/peak: Int %`, `minutes: Int`, `workoutType: String`, `userGoal`, `fridgeItems`, `cookingTimeMinutes`, `userMessage`, raw `currentSteps`, and the user's Arabic free text.
- **Destination:** `CAPTAIN_ARABIC_API_URL` (Info.plist / environment) — HTTP POST `{"text": userInput}`. Falls back to `http://localhost:3000/captain-ar` on Simulator per [line 323-327](AiQo/Features/Captain/CaptainIntelligenceManager.swift:323).
- **Sanitizer status:** **none**. `generateArabicAPIReply` does not touch `PrivacySanitizer` at any step. It does call `AICloudConsentGate.requireConsent()` at [line 275](AiQo/Features/Captain/CaptainIntelligenceManager.swift:275).
- **User consent required:** yes (AICloudConsentGate). Consent is persisted in UserDefaults and never re-verified.
- **Fix vector:** Wrap the input at entry to `generateCaptainResponse` (or inside `generateArabicAPIReply`) with a new `PrivacySanitizer.sanitizePromptForCloud(text:)` helper that runs `sanitizeText` plus numeric bucketing for vitals in free text.

### Leak #2 — Workout summary prompt (Arabic)
- **Location:** [AiQo/Services/Notifications/NotificationService.swift:850-909](AiQo/Services/Notifications/NotificationService.swift:850) (prompt builder) → [NotificationService.swift:629](AiQo/Services/Notifications/NotificationService.swift:629) (invocation)
- **Trigger:** Any completed workout observed by `HKObserverQuery` on `HKObjectType.workoutType()` ([NotificationService.swift:656-669](AiQo/Services/Notifications/NotificationService.swift:656)) OR a live workout end dispatched via `CaptainSmartNotificationService.handleWorkoutCompleted` ([NotificationService.swift:223-235](AiQo/Services/Notifications/NotificationService.swift:223)). `handleWorkoutEnded` runs an anchored sync of every new HKWorkout at startup.
- **Data leaked:** embedded in the prompt template at [lines 874-885](AiQo/Services/Notifications/NotificationService.swift:874) (Arabic) and [lines 897-908](AiQo/Services/Notifications/NotificationService.swift:897) (English):
  - `workoutType: String` — raw HK activity name
  - `minutes: Int` — rounded duration
  - `calories: Int` — `keyMetrics["calories"]`
  - `averageHR: Int` — from `statistics.averageQuantity` of `HKQuantityType.heartRate`
  - `distance: String` — `%.2f` km
  - `below/zone2/peak: Int %` — percentage of time in HR zone buckets computed from raw `HKQuantitySample` HR samples ([lines 736-788](AiQo/Services/Notifications/NotificationService.swift:736)).
- **Destination:** Arabic branch → `generateArabicAPIReply` → `CAPTAIN_ARABIC_API_URL`. English branch → on-device Foundation Models (safe).
- **Sanitizer status:** **none**. No PII redaction, no bucketing, no name removal.
- **User consent required:** yes (via `AICloudConsentGate` inside CIM's Arabic path).
- **Fix vector:** Two layers — (a) bucket numbers before interpolation (round HR to 5 bpm, calories to 10 kcal, distance to 0.1 km, zone% to 5% bins), OR (b) REPLACE the LLM call with the existing deterministic `fallbackMessage` ([lines 911-938](AiQo/Services/Notifications/NotificationService.swift:911)) which already produces solid Arabic/English 20-word summaries.

### Leak #3 — Workout summary prompt (English)

### Leak #13 — CaptainContextData fed to CloudBrainService (sanitized, but narrow)
- **Location:** [AiQo/Features/Captain/CaptainContextBuilder.swift:214-225](AiQo/Features/Captain/CaptainContextBuilder.swift:214) → `CaptainViewModel.processMessage` → `BrainOrchestrator.processMessage` → `CloudBrainService.generateReply` → `sanitizer.sanitizeForCloud(...)`.
- **Sanitizer status:** YES, but narrow. `sanitizeHealthContext` ([PrivacySanitizer.swift:293-300](AiQo/Features/Captain/PrivacySanitizer.swift:293)) buckets only `steps` and `calories`. `CaptainContextData` also carries `sleepHours: Double`, `heartRate: Int?`, `timeOfDay`, `toneHint`, `stageTitle`, `bioPhase`, `emotionalState`, `trendSnapshot`, `messageSentiment`, `recentInteractions` — but **these fields are stripped entirely** when the sanitizer rebuilds `CaptainContextData` without them (positional `CaptainContextData(steps:calories:vibe:level:)` drops everything else to the default values). So the sanitizer is actually *protective here by omission* — but it silently discards Brain V2 signals on the cloud path.
- **User consent required:** yes.
- **Fix vector:** Not a leak. But document this behavior — Brain V2 features like emotional state and trend snapshot are dropped on the cloud path and would need explicit bucketing/masking to be safely forwarded.

### 4.2 Gaps

**HealthKit numeric precision — NOT bucketed:**
- `heartRate: Int?` — `CaptainContextData.heartRate` exists but `sanitizeHealthContext` doesn't bucket it (and rebuilds `CaptainContextData` without this field entirely — see §3 Leak #13).
- `sleepHours: Double` — same. Dropped by the sanitizer on the cloud path.
- HRV / SDNN / RHR / VO2Max — no representation in `CaptainContextData`; would need schema extension to be handleable.
- Workout distance (meters/km), duration minutes, HR-zone percentages, workout type — no handling; `NotificationService.buildPrompt` interpolates all of these as raw text.
- Resting heart rate — not bucketed.
- Active calories vs basal calories — only `calories` (appears to be active) is bucketed.
- Workout GPS routes (latitude/longitude series from `HKWorkoutRoute`) — not represented and never sanitized.

**PII types — gaps:**
- First names *not* passed via `knownUserName` — e.g. a message `"my wife Sarah called"` keeps "Sarah" because the `replaceSelfIdentifyingPhrases` rules only catch first-person disclosures. The `replaceExplicitProfileFields` rules only catch labelled fields.
- Arabic names without the `اسمي` prefix leak.
- Last names / surnames in free text — no entity detection.
- Apple IDs / iCloud handles — not specifically targeted (may be caught by email regex if formatted as email).
- Device identifiers (IMEI, serial numbers, advertising IDs) — not explicitly handled.
- Phone numbers shorter than 8 digits and without `+` prefix — the regex pattern `(?<!\d)\+?\d(?:[\d\-\s()]{6,18})\d(?!\d)` requires at least 8 total digits.
- IBANs and credit card numbers longer than 32 characters — the numeric-sequence rule is bounded to 30.

**Image metadata:**
- **Only kitchen images** are EXIF-stripped. `sanitizeForCloud` ([line 106-108](AiQo/Features/Captain/PrivacySanitizer.swift:106)) applies `sanitizeKitchenImageData` **only when `request.screenContext == .kitchen`**. Any future attachment from `.gym`, `.peaks`, `.myVibe`, `.mainChat`, or `.sleepAnalysis` would forward raw EXIF/GPS.
- **Not relevant today** because no other screen context attaches image data, but it's a one-route-flip risk.

**Location / GPS:**
- No latitude/longitude regex.
- No workout route sanitization.
- Addresses and city names in free text are not detected.

**Conversation:**
- Truncation to 4 messages is aggressive — good for privacy, may harm context.
- Truncated messages may have already leaked numbers; bucketing would need to run *inside* `sanitizeText` not just on `CaptainContextData`.

**Cloud-safe memories:**
- `cloudSafeMemories` passed in *as trusted* — the sanitizer trusts `MemoryStore.buildCloudSafeRelevantContext` to produce already-safe text. Not audited here. If any memory entry contains raw HK numbers, they leak.

**`userProfileSummary`:**
- `sanitizeForCloud` discards this field unconditionally (writes empty string at [line 118](AiQo/Features/Captain/PrivacySanitizer.swift:118)). Good — this is the aggressive default. `CaptainBackgroundNotificationComposer.buildPersonalizationPreamble` also builds a profile summary with bedtime/wake-time, but this is consumed by `LocalBrainRequest` which doesn't pass through `sanitizeForCloud`.

### 7.3 Gaps and concerns

1. **Consent is forever.** Once granted, consent is stored in UserDefaults under `aiqo.aiDataConsent.accepted` with no expiry ([AIDataConsentManager.swift:58-65](AiQo/Services/Permissions/AIDataConsentManager.swift:58)). No re-consent prompt on legal/policy updates, app reinstall, or model changes. If Apple or GDPR tighten rules, existing users keep the old consent.
2. **Legacy migration is silent.** `migrateLegacyKeysIfNeeded` ([line 79-88](AiQo/Services/Permissions/AIDataConsentManager.swift:79)) promotes `aiqo.ai_data_consent.granted` → `aiqo.aiDataConsent.accepted` without re-asking the user. An old consent from a prior app version is honored unchanged.
3. **Background task consent path.** `ensureConsent(presentIfPossible:)` only presents a sheet when `UIApplication.shared.applicationState == .active` ([line 47](AiQo/Services/Permissions/AIDataConsentManager.swift:47)). In a background task, the sheet is suppressed, and the check returns `false` — which then throws `.consentRequired` and aborts the LLM call. That's *correct* behavior, but means a user who has never granted consent will silently see no inactivity nudges translated via Gemini — only the deterministic fallback at `finalBody = defaultArabicMessage` ([SmartNotificationScheduler.swift:530](AiQo/Core/SmartNotificationScheduler.swift:530)).
4. **No per-destination consent.** A single `hasUserConsented` flag covers both Gemini (CloudBrainService, Translator, VoiceAPI) and `CAPTAIN_ARABIC_API_URL` (a different endpoint, potentially in a different jurisdiction). The consent text shown in `AIDataConsentView` ([line 91-175](AiQo/Services/Permissions/AIDataConsentManager.swift:91)) is generic. For App Store review, this may be flagged — data destinations should be disclosed individually.
5. **Consent does not gate HealthKit READ.** Neither `AICloudConsentGate` nor `AIDataConsentManager` gate `HKHealthStore.requestAuthorization`. Reading is governed only by HealthKit's native permission. This is correct Apple model.
6. **Consent is not checked inside `sanitizeForCloud`.** Defense-in-depth note: the sanitizer is purely data-transformation. If a caller forgot to check consent, the sanitizer would still run and ship data. The consent gate lives in the callers, which makes adding a new caller (and forgetting the gate) a real foot-gun.
7. **MemoryStore cloud-safe context is NOT behind explicit consent.** `MemoryStore.buildCloudSafeRelevantContext` ([MemoryStore.swift:194](AiQo/Core/MemoryStore.swift:194)) is called from `CloudBrainService.generateReply` ([line 52](AiQo/Features/Captain/CloudBrainService.swift:52)) — consent is checked at line 44 before this, so covered. But if another caller ever calls `buildCloudSafeRelevantContext` and forwards to cloud, there's no re-check.
8. **Developer nudge test is consent-gated downstream.** `queueDeveloperTestCoachNudge` doesn't check consent itself but calls `translateToFriendlyIraqiArabic` which hits `CoachBrainLLMTranslator.translate` (consent gated). So a dev nudge cannot silently fire an API call on a non-consented user. ✓

## 9. Open Questions

1. **`CAPTAIN_ARABIC_API_URL` destination jurisdiction.** The consent sheet ([AIDataUseDisclosure.swift:74](AiQo/Features/Compliance/AIDataUseDisclosure.swift:74)) was not inspected for per-destination disclosure. If the Arabic API server is self-hosted in a different country than Google's Gemini endpoint, App Review may require separate consent. *Action:* audit disclosure text in `ai.consent.*` `.strings` entries.
2. **`CaptainContextBuilder.systemPrefix` consumers.** The `[SYSTEM CONTEXT: Stage N, ..., Steps: X, Sleep: Y.Yh, Calories: Z, Heart Rate: W]` prefix at [CaptainContextBuilder.swift:184-189](AiQo/Features/Captain/CaptainContextBuilder.swift:184) could leak if any caller forwards it to Gemini. 3. **`MemoryStore.buildCloudSafeRelevantContext` audit.** This function returns a "cloud-safe" string injected into `workingMemorySummary`. The recon did not inspect the implementation; if any memory entry contains raw HK numbers or user names, they leak. *Action for P0.2 or P0.3:* read [MemoryStore.swift:194](AiQo/Core/MemoryStore.swift:194) and verify bucketing/redaction.
4. **`ProactiveEngine.shared.evaluate` content origin.** `SmartNotificationScheduler` falls back to the translator path only if ProactiveEngine returns no decision. Inspecting `ProactiveEngine` would confirm whether its `.sendNotification(content, ...)` content is deterministic or draws from any cloud LLM. Not inspected in this recon.
5. **Arabic-language detection reliability.** `CaptainIntelligenceManager.textContainsArabic` ([line 598-612](AiQo/Features/Captain/CaptainIntelligenceManager.swift:598)) uses Unicode block checks. A mixed-language message with one Arabic character will take the Arabic route. Fine for correctness, but means almost any Captain message from a UAE-based user lands on the Arabic endpoint regardless of the system language setting.
6. **Background delivery on workout observer.** The `HKObserverQuery` on `HKObjectType.workoutType()` at [NotificationService.swift:656-669](AiQo/Services/Notifications/NotificationService.swift:656) — does it run with `enableBackgroundDelivery`? (Not grep-verified here.) If yes, workout summary LLM calls can fire while the app is suspended, further underscoring why Leak #2 deserves REPLACE over WRAP.
7. **Apple Review evidence.** The previous audit flagged Section 6.7 (Apple Compliance Risks). This recon confirms the primary concern: `CAPTAIN_ARABIC_API_URL` is a **second cloud destination** beyond Gemini that Apple reviewers may not see disclosed in the single `AIDataConsentView`. Requires disclosure update before next submission.
8. **`CoachBrainMiddleware.sanitizer` is instantiated but unused.** [CoachBrainMiddleware.swift:241](AiQo/Features/Captain/CoachBrainMiddleware.swift:241) stores a `PrivacySanitizer()` but no grep hit for `sanitizer.` within the file. Either (a) it was meant to wrap `translator.translate` inputs and the wiring was forgotten, or (b) it's dead code. *Action:* confirm and either wire it up (P0.2) or delete (hygiene).
